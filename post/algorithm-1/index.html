（粗略分析：假设n个操作序列每个操作都是代价最大的操作的最坏情况；
精细分析：代价最大的操作可能只出现少数次）

如果粗糙地分析，每个Multipop最坏是 O(n)O(n)O(n) ，最坏情况是n个序列都是Multipop，这样的话 T(n)=O(n2)T(n)=O(n^2)T(n)=O(n2) ，平摊代价 T(n)/n=O(n)T(n)/n=O(n)T(n)/n=O(n) 。

对于长度为n的序列，最多有n次push（其实n-1也差不多，我们这里看阶数即可），从而最多有n次pop。因此，n个栈操作序列的代价 T(n)≤2n=O(n)T(n) \le 2n = O(n)T(n)≤2n=O(n) 。则平摊代价 T(n)/n=O(1)T(n)/n = O(1)T(n)/n=O(1) 。
